# Pilot Tracker

Внутрішня продуктова документація для вузького B2B-ish напряму: продукт для соло-фаундерів і малих founder-led команд, яким потрібно тримати під контролем `design partners`, `paid pilots`, `promises`, `success criteria`, `renewal dates`, `next steps`, `blockers` і `pilot risk`.

Поточна стратегія не передбачає негайну розробку повного продукту. На цьому етапі фокус такий:

- landing page
- збір фідбеку
- discovery calls
- paid validation pilot
- рішення про MVP тільки після сигналу ринку

Головна точка входу в документацію: [docs/DOCS_INDEX_UA.md](docs/DOCS_INDEX_UA.md)

## Що є в репозиторії

- `app/`, `components/`, `lib/` - поточний код лендингу та форми збору лідів
- `docs/` - актуальна продуктова документація українською

## Як читати документацію

1. Почати з [docs/DOCS_INDEX_UA.md](docs/DOCS_INDEX_UA.md)
2. Потім прочитати [docs/PRODUCT_OVERVIEW_UA.md](docs/PRODUCT_OVERVIEW_UA.md)
3. Далі перейти до стратегії, ICP, плану валідації та roadmap

## Важливе обмеження

Документація навмисно не роздуває продукт до CRM, customer success suite або AI copilot. Вона фіксує вузьку гіпотезу і дисципліну прийняття рішень до початку повноцінної розробки.
